Tutorial de Git básico

02/02/2024 Luis Fernando Rodríguez

Git es el sistema de control de versiones por excelencia. Desde su lanzamiento en 2007 ha sabido imponerse sobre las demás alternativas y actualmente es una herramienta muy importante para los desarrolladores. Tener una buena gestión de versiones en un proyecto puede ayudar enormemente a simplificar el proceso de desarrollo y en este artículo voy a enseñar todo lo básico para manejar repositorios de manera local, para ir paso a paso no voy a hablar sobre plataformas que utilizan git como Github o Gitlab en este artículo.

1. Instalación de Git

Para instalar Git ve a la página oficial de Git y sigue las instrucciones para realizar la instalación en su sistema operativo.

2. Configuración de Git

Una vez instalado Git es necesario indicar quién es la persona que realiza los cambios en el equipo. Para lograr esto, digita los siguientes comandos en la terminal de su sistema operativo, cambiando los valores de nombre y algo@email.com por su nombre y correo electrónico, respectivamente.

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

3. Inicializar un repositorio de Git

Ahora sí ya está todo listo para iniciar el primer repositorio. Simplemente hay que navegar en la terminal hasta llegar a la carpeta raíz del proyecto al que queramos manejar sus versiones. Para esto, basta con digitar el siguiente comando:

git init

4. Realizar cambios y guardarlos

Para realizar cambios primero hay que agregar archivos al área de preparación. Para esto, basta con digitar el siguiente comando:

git add nombre_del_archivo

¿Entonces tengo que ir agregando archivos uno a uno? No, puede digitar el siguiente comando si desea que todos los archivos y carpetas se agreguen (hay una excepción y serían los archivos señalizados en .gitignore). Digite el siguiente comando:

git add .

Una vez que los archivos y carpetas están correctamente agregados en el área de preparación, ya es posible guardarlos haciendo un commit con el siguiente comando. Es recomendable agregar un comentario cada vez que guarda cambios con el fin de identificar con mayor facilidad qué cambios hizo esa versión del proyecto.

5. Ver historial de cambios

Basta con colocar el siguiente comando, mostrará la lista de commits con información como autor, fecha y descripción:

git log

6. ¿Qué son las ramas y cómo se trabaja con ellas?

Una rama es como una bifurcación en el proyecto (una especie de universo paralelo), aunque suene un poco complejo no lo es tanto. Imagina un árbol, este tiene un tronco que sería la rama principal y a partir de este tronco pueden salir ramas que toman como base al tronco. Algo así son las ramas en Git, versiones del proyecto que toman la base de este pero que se pueden alejar y tomar su propio rumbo. Para crear una rama hay que digitar el siguiente comando:

git branch nombre_de_la_rama

Para cambiar de rama se utiliza el siguiente comando:

git checkout nombre_de_la_rama

Ahora podría trabajar en una rama nueva para que los cambios que realice no afecten a la rama principal. Sin embargo, si en algún momento desea fusionar las ramas puede hacerlo con los siguientes comandos:

git checkout rama_destino
git merge rama_a_fusionar

Conclusión

Felicidades, has completado el tutorial básico de Git. En este artículo me limité a únicamente hablar sobre cómo trabajar con repositorios locales. Sin embargo, con Git se puede trabajar en repositorios remotos para colaborar con más personas. Probablemente lo veamos en un futuro artículo.
